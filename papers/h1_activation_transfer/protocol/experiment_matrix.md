## Alpha Grid

Pilot alpha values:

- `0.05`
- `0.10`
- `0.20`
- `0.30`
- `0.40`
- `0.50`
- `0.70`
- `0.90`
- `1.00`

After pilot, choose:

- best alpha
- one lower neighbor
- one upper neighbor

Run those with all final seeds.

## Output Naming

Suggested pattern:

```text
papers/h1_activation_transfer/results/final/
  h1_{condition}_{direction}_seed{seed}.json
```

For deterministic conditions:

```text
h1_{condition}_{direction}_deterministic.json
```

The unified runner implements this pattern and adds an alpha suffix when a
scale-corrected condition uses a non-default alpha value.

## Promotion Rule

An experiment can enter a main paper table only if:

1. It uses `clean_eval.jsonl`.
2. It follows `final_protocol.md`.
3. It has per-sample outputs.
4. Its metrics are generated by the common summarization script.
5. Its condition is listed in this matrix or in a recorded amendment.
