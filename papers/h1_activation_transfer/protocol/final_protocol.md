# Final Protocol for H1 Clean Rerun

> Status: draft. This file must be frozen before any final clean rerun is treated
> as paper evidence.

## Research Question

Can cross-model activation transfer outperform natural-language relay for
model-to-model communication?

In this paper, "activation transfer" means:

1. Extract sequence-level hidden states from a sender model.
2. Translate them into the receiver model's hidden-state space.
3. Inject translated activations into the receiver during prompt prefill.
4. Evaluate downstream task accuracy.

## Claim Boundary

The paper should only claim results for the tested setting:

- Pythia-family models.
- Multi-hop reasoning.
- Single-layer inference-time injection.
- 67 percent relative depth unless a later protocol amendment adds layer scans.
- Linear or explicitly specified translation-layer variants.

The paper must not claim that all activation-based model communication is
impossible.

## Input Format

For activation-transfer conditions, model A and model B receive the same full
input during prompt prefill:

```text
Context: {context}
Q: {question}
A:
```

This keeps sequence lengths aligned for replacement-style injection.

For natural-language relay, model A generates a relay text from the source
context, then model B receives:

```text
{relay}
Q: {question}
A:
```

Because these two conditions expose model B to different information formats,
the paper must report both absolute accuracy and delta relative to each
condition's own baseline.

## Layers

Default relative depth:

- 67 percent

Expected Pythia layer mapping:

- Pythia-160M: layer 8 of 12
- Pythia-410M: layer 16 of 24

The exact resolved layer names must be stored in every result JSON.

## Translation Layer

Primary translation layer:

- Linear map from sender hidden dimension to receiver hidden dimension.

Training target:

- L2-normalized target activations unless a raw-space variant is explicitly
  registered in `experiment_matrix.md`.

Reported translation-quality metrics:

- `R2(norm)`
- `cosine_similarity(norm)`
- `R2(raw)`
- translated output norm distribution
- receiver hidden-state norm distribution

## Injection Modes

The final rerun should include:

1. No injection.
2. Natural-language relay.
3. Additive sequence-level injection.
4. Uncorrected replacement.
5. Scale-corrected replacement.
6. Best-alpha scale-corrected interpolation.
7. B to B self-injection control.
8. Same-norm random replacement control.
9a. Full-n same-length shuffled-pair translation control.
9b. Strict same-length no-self shuffled-pair translation control.

Scale-corrected replacement is:

```text
corrected = normalize(translated) * norm(original_receiver_hidden)
new_hidden = alpha * corrected + (1 - alpha) * original_receiver_hidden
```

The shuffled controls are defined as follows:

- `shuffled_translation`: keeps all `n=396` clean-eval samples and shuffles
  translated activations only within identical prompt-length buckets. If a
  bucket has one sample, the sample maps to itself and the run must report
  `shuffle_self_fallback_count`. When this count is nonzero, this condition is
  diagnostic or appendix evidence rather than the cleanest shuffled control.
- `shuffled_translation_strict_matched`: uses the same same-length bucket rule
  but drops singleton prompt-length buckets before shuffling. It must have
  `shuffle_self_fallback_count=0`. For the frozen clean-eval hash in this
  protocol, this evaluates `n=383` samples and records 13 excluded singleton
  samples.

Default seeds:

- `42`

## Decoding

Default generation parameters:

- greedy decoding
- `do_sample = false`
- `max_new_tokens = 64` for receiver answers
- `max_new_tokens = 128` for natural-language sender relay text
- `repetition_penalty = 1.3`
- `no_repeat_ngram_size = 3`

All generation parameters must be serialized into result JSON files.

## Frozen Runner Defaults

These defaults are part of the final protocol. Changing any value requires a
protocol amendment and a new validation report.

| Field | Frozen value | Reason |
| --- | --- | --- |
| `max_length` | `256` | Controls tokenization truncation and replacement-style sequence alignment. |
| `layer_rel` | `0.67` | Resolves to `gpt_neox.layers.8` -> `gpt_neox.layers.16` for the primary direction. |
| `injection_scale` | `0.01` | Low-strength additive injection setting inherited from the corrected M6 sequence-level rerun. |
| `alpha` for `scale_corrected` | `1.0` | Full scale-corrected replacement. |
| `best_alpha` | `0.30` | Selected from the historical M7 alpha pilot before the clean-eval final rerun; not selected by post-hoc tuning on clean-eval. |
| `shuffle_injection_mode` | `scale_corrected` | Tests sample pairing while preserving the receiver hidden-state norm. |
| receiver `max_new_tokens` | `64` | Answer generation budget. |
| sender relay `max_new_tokens` | `128` | Natural-language relay generation budget. |
| clean-eval provenance path separator | Windows-style backslash (`\`) | The original frozen `clean_eval_sha256` was first computed on a Windows host where `pathlib.Path.__str__` rendered `\` as the path separator inside per-row `source_eval_file` provenance fields. The clean-eval builder now normalizes that field to backslashes on every platform via `stable_provenance_path()`, so the SHA-256 hash is byte-identical when rebuilt on Linux CI. Any future rebuild on a fresh source tree must keep this normalization to preserve the locked hash. |

## Result JSON Requirements

Every final result file must include:

- condition name
- model A and model B identifiers
- layer names
- seed
- clean evaluation file path
- clean evaluation hash
- decoding parameters
- injection mode
- translation checkpoint path, if applicable
- aggregate metrics
- per-sample predictions and metric outcomes

Result files without per-sample outputs cannot enter main paper tables.

## Execution Entry Points

Clean reruns should use the unified runner:

```text
python -m rosetta.experiments.phase1.h1_final_runner
```

Default behavior:

- reads `data/tasks/multi_hop_reasoning/clean_eval.jsonl`
- writes per-condition JSON files under `papers/h1_activation_transfer/results/final/`
- uses the final full-input prompt for activation-transfer and no-inject conditions
- uses actual sender-generated relay text for the natural-language relay condition
- records clean-eval hash, layers, generation settings, checkpoint path, aggregate metrics, diagnostics, and per-sample predictions

The runner intentionally owns the prompt tokenization path for injection
conditions.  Replacement-style injection must use the same `max_length` for
sender extraction and receiver generation, and the runner records token and
sequence-length agreement for each sample.

Paper tables should be generated by:

```text
python papers/h1_activation_transfer/scripts/summarize_results.py
```

The summarizer recomputes final paper metrics from per-sample predictions rather
than trusting copied aggregate numbers.
